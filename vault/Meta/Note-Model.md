# Note Model

Use this structure for new vault notes.

## Standard sections

- purpose
- what this note covers
- what this note does not cover
- canonical sources
- related notes

## Metadata

- use frontmatter for type, status, aliases, and tags when that helps discovery
- keep the title specific enough that the note can stay narrow
- prefer one strong note over a folder full of overlapping notes

## Quality checks

- can a contributor tell at a glance what problem the note solves
- does the note point back to the canonical source instead of repeating it
- does the note avoid mixing unrelated topics
