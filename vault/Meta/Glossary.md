# Glossary

This note translates project-specific terms into a shared vocabulary.

## Terms

- abi-data: the Firestore database ID used by the app
- MOC: map of content, a vault index note
- canonical source: the file that should settle the exact fact
- rules-first: access is enforced by Firestore and Storage rules, not UI assumptions
- dashboard subdomain: the authenticated app surface
- vault: the Obsidian-style handbook for this repository

## Use this note when

- a term appears in one note but not another
- a contributor needs the repo language decoded quickly
- a note should explain an abbreviation instead of reusing it blindly
