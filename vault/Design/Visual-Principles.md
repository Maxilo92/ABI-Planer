# Visual Principles

The UI should look intentionally designed, not autogenerated.

## Direction
- choose a clear visual identity and keep it consistent
- use a restrained palette with one strong Akzent, not default purple-on-white gradients
- prefer purposeful typography over generic system-default styling
- use atmosphere with texture, depth, or subtle pattern instead of flat emptiness
- make important states obvious: active, disabled, loading, success, warning, archived
- prefer wording with Umlaute in visible UI copy when that fits the language of the product

## What to avoid
- repetitive dashboard card grids or tile walls with no hierarchy
- overly rounded, generic SaaS surfaces everywhere
- decorative motion that does not communicate state
- empty layouts that rely on filler content

## Design intent
- the interface should feel calm, clear, and credible
- each screen should have one primary action and one obvious reading order
- visual treatment should support trust, not novelty for its own sake
