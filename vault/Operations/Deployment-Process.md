# Deployment Process

## Baseline flow
1. Make the change in a focused branch or working tree.
2. Validate the touched area.
3. Update the handbook if the change alters working assumptions.
4. Update `VERSION` and `CHANGELOG.md` when the repo release state changes.
5. Deploy only through the approved release path for the project.
