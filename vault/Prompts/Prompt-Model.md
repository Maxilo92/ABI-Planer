# Prompt Model

Use this structure when writing a new prompt for the project.

## Standard shape

- goal
- context
- constraints
- expected output
- validation

## Rules

- keep the prompt specific enough that it can be reused
- name the files or systems the prompt should touch
- state whether the answer should be a plan, an implementation, or a review
- prefer one strong prompt over a generic catch-all

## Good prompt qualities

- short enough to scan fast
- explicit about scope boundaries
- aligned with the vault-first workflow
