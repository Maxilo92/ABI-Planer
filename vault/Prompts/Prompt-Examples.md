# Prompt Examples

These examples are meant to live in the vault and give contributors a direct starting point.

## Implementation prompt

"Implement the smallest change needed to add a new settings section for push notifications. Touch only the relevant UI, hook, and schema files. Validate the affected files after the edit."

## Review prompt

"Review the vault changes for missing links, broken note structure, and misplaced content. Focus on behavior regressions in navigation or documentation flow."

## Research prompt

"Map the cross-cutting system path for this feature: start at the user-facing surface, then trace the data, rules, and server-side processing, and end with the canonical source files."

## Cleanup prompt

"Normalize the markdown structure of the touched vault notes without changing meaning. Keep headings, lists, and links consistent and preserve the current information architecture."
