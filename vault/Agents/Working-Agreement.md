# Working Agreement

This note defines the baseline behavior for anyone working in the repository through this vault.

## Rules

1. Start with the smallest relevant MOC and the canonical source note.
2. Keep one note focused on one topic.
3. Prefer links over duplication.
4. If a fact must be exact, point to the canonical file outside the vault.
5. After an edit, validate the smallest thing that can prove the change is sound.
6. Do not use destructive recovery steps when a local merge or patch will do.
7. Update the vault when the repo changes in a way that affects working assumptions.
