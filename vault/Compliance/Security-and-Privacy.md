# Security and Privacy

## Core expectations
- treat Firestore and Storage rules as the source of truth for access control
- keep secrets out of the client bundle
- keep approvals, roles, and domain restrictions aligned with the canonical security docs
- review any auth, profile, logging, or data-retention change as a compliance-sensitive change
